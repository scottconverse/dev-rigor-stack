dev-rigor-stack

A six-gate delivery discipline for AI coding agents — so "done" means proven, not "looks right."

View on GitHub Read the manual

What it is

AI coding agents are confident, and confidence isn't correctness. An agent says "done" when it means "I wrote code that looks right" — a different claim from "I proved it runs," different again from "I proved it's correct." This stack is a discipline that closes that gap: every change is exercised the way a real user hits it before anyone calls it finished, and the call to make a release real stays with a human.

The loop

Every unit of work walks the same line. Nothing advances on a claim — only on evidence.

PLAN→ BUILD→ VERIFY→ REVIEW→ MERGE

Then, once per version, a release gate: the full gauntlet driven to zero findings, the docs checked for overclaiming, a rollback plan named — and the owner's go/no-go on the tag.

Get started

As Claude Code skills:

git clone https://github.com/scottconverse/dev-rigor-stack
cd dev-rigor-stack
./install.sh                     # macOS / Linux / Git Bash  (./install.ps1 on Windows)
./install.sh --with-ponytail     # also add the optional code-minimalism lane
./install.sh --target ~/.codex/skills   # install elsewhere, e.g. Codex

Inside a Cowork or Codex session (no terminal): just ask the agent to install it from the repo — it copies the skills into the right place and can offer the ponytail lane as a yes/no.

For any other agent (ChatGPT, Gemini, Codex): run ./export/export-portable.sh and paste the generated portable-bundle.md into its system prompt. Claude-specific mechanics read as plain guidance — nothing is diluted from the Claude side to serve the export.

Honest scope

What it is

A repeatable discipline that makes an agent prove its work at the layer of the claim. Model-agnostic and tool-agnostic. A way to keep long, multi-machine efforts from losing their decisions.

What it isn't

Not magic, and not a guarantee of bug-free code. It's deliberate overhead that only pays off when being wrong is expensive — so it tells you, in every gate, when to skip it.
